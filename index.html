numpy,pandas and matplotlib

​ numpy, pandas 和matplotlib是Python中用于数据分析和画图的几个非常重要的库。想要用Python进行数据分析必须要熟练掌握这几个库，同时机器学习和人工智能算法中也会经常以这些库作为基础，因为都和数据脱不了关系。下面是我对这几个库的一些基础知识的简单总结。

1.numpy

numpy提供一个叫做ndarray的数据类型，意思是n维数组(n dimension array)。与Python自带的列表不同，这种数据类型的结构要高效许多。导包的方法是

创建一个ndarray

常规是直接使用np.array()方法，返回的就是一个ndarray类型，显示的格式是array(list格式)如：输入np.array([1,2,3])，返回的结果是array([1,2,3]),为ndarray类型。注意np.array()传入的参数要是一个列表。

如果要创建一个特殊的ndarray矩阵，就要用到np自带的函数来创建。这些函数与MATLAB中的矩阵函数几乎一模一样。比如np.ones(), np.zeros(),np.full()等。内置的变量有shape和dtype等

如： np.ones(shape=(3,4),dtype=np.float32)返回一个三行四列数据的类型为float32的矩阵

np.full()里面多了一个参数，要输入填充的值：np.ones(shape=(2,3),fill_value=6)返回一个两行三列数值全为6的矩阵。

还有两个用来分段的函数np.linspace()和np.arange(),基本的用法是np.linspace(start,stop,num),还可以设置是否开区间还是闭区间。np.arrage(start,stop,step)期中step为步长。两者的作用和用法都差不多。

还有一些在统计学中的函数

ndarray的四个属性

ndarray具有四个属性，分别是ndim(维度)，shape(形状),size(总长度),和dtype(元素类型)

ndarray的基本操作

索引：n[1,2,3]表示取出第二个矩阵的第三行四列中的元素，索引下标是从0开始的，逗号是维度的分割。根据索引也可以进行修改能操作。

切片：切片也是按照维度进行切片，如n[0:1,1:2,2:3],将数据进行翻转可用n[:,:,::-1]等双冒号后面加-1的方式。

变形：用reshape方法，n.reshape((6,4))把一个三维的ndarray改成一个二维六行四列的ndarray矩阵。reshape要求元素的总数保持一致。注意reshape中传入的是一个元组。

这里因为两个矩阵列数相同行不同所以只能进行垂直级联

在MATLAB中使用Python

在B站上看到了Python与MATLAB交互的视频，记录一下方法。

总的来说，matlab中直接在前面加上py.就可以调用Python的所有用法。

在MATLAB中使用Python的module

当写好一个Python脚本之后，想要在MATLAB里运行有几种方法

1.直接导入：

​ mod = py.importlib.import_module(module名)

2.重新导入：

​ py.importlib.reload(module名)

3.将当前文件夹加入到Python搜索路径：

​ if count(py.sys.path,’’) == 0

​ insert(py.sys.path,int32(0),’’);

​ end

当Python脚本经过修改之后要重新导入才能执行修改后的脚本。

这个可能存在版本的限制，老版本的MATLAB不一定能实现。

关于hexo上传图片的显示问题

​ 一直被图片显示的问题困扰，昨晚终于算是解决了这个问题，虽然不是个完美的解决方案。

​ 第一次上传博客就发现了这个问题，只要本地图片上传就无法显示，而用Markdown语法直接复制图片链接上传之后则可以显示，相当于采用外链的方式。网上有很多解决本地图片上传无法显示的帖子，都尝试后发现还是无法解决。无奈只能本地图片也考虑采用外链引用的方式，我先把图片上传到微博，然后直接在微博上复制图片的链接，发现还是不行。

​ 再查资料后发现，微博等网站有限制外链，解决方式是在blog中的head.ejs文件中加入一行

<meta name="referrer" content="no-referrer" />

​ 本人的路径是

​ blog\themes\landscape\layout_partial\head.ejs

​ 路径基本上都是大同小异。

​ 原理大致是http请求的header中有referrer字段，表示请求的原地址信息，服务器通过referrer来判断请求是否来自本站，若不是就会拦截。把referrer改成no-referrer发送请求时就不会带上referrer使对方服务器无法拦截。
