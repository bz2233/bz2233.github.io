C++常用STL之map

C++中常用的映射是map,映射只能多对一不能一对多。有点像Python里面的字典类型，有key和value。

引用

#include

using namespace std;

构造一个映射

map<T1,T2> m; //定义一个名为m的从T1类型到T2类型的映射

插入元素

插入元素需要插入一个新的映射，需要引入头文件utility

如果插入的key已经存在，则不会插入新的value代替原来的value，也就是这次插入是无效的。

访问映射可以用中括号的方式访问或者更改，这点也跟Python的字典一样

dict[“tom”]则会返回1

遍历映射也需要通过迭代器

map<string,int>::iterator it；

在遍历过程中map也是按照关键字从大到小遍历的，跟set一样

练习：

给定一串数字，统计出现次数最多的数字是多少，如果有出现次数一样多的情况则按照最大的数字输出。

不用map，首先将字符串排序之后统计每一个数连续出现了多少。

用map,先定义一个由int到int的 map，第一个int表示出现的数字，第二个表示出现该数字的个数。

C++常用STL之set

C++中常用的集合是set

引用库

#include

#using namespace std;

定义集合

set s //定义一个名为s、存储类型为T类型数据的集合

插入元素

s.insert(value) //

注意如果集合中已经存在了某个元素，再次插入不会产生效果，集合中不会出现重复元素

删除元素

s.erase(value) //如果集合中不存在这个元素，不进行任何操作

判断元素是否存在

s.count(value) //如果存在则返回1，不存在返回0

返回元素个数

s.size()

遍历集合

遍历set需要使用迭代器，固定格式为

for (set::iterator it = s.begin(); it != s.end(); it++){
cout << *it << “ “;
}

当set配合结构体来使用时，系统不知道结构体比较大小的方式额，而set是需要经过排序的，所以我们需要一种方式来告诉系统怎么比较结构体的大小。其中有一种方法叫运算符重载，需要重新定义小于符号。

bool operator<(const struct_name &rhs) const

输入

6
5 6
1 2
2 1
3 4
1 2
1 1

输出

1 1
1 2
2 1
3 4
5 6

可以看到经过遍历之后是已经自动排好序了。

using namespace std;

定义动态数组

vector vec; //定义一个名为vec的存储T类型的动态数组

插入元素

vec.push_back(value); //在数组最后插入一个新的元素

获取长度和访问元素使用vec.size() 用[]访问vector中的元素

for (int i = 0;i < vec.size();i++){
cout << vec[i] << endl;
}

修改元素可以通过直接赋值的方法

删除元素用vec.pop_back()删除数组中最后一个元素

vec.pop_back()

51单片机的系统指令

51单片机的指令格式：
操作数 目的操作数,源操作数 ；注释

():用于注释中表示存储单元的内容，即括号内的地址所对应的内容

2.1.1 立即寻址

直接给出操作数的寻址方称为立即寻址。立即数用前面加“#”号的8位数或16位数表示。

2.1.2 直接寻址

指令中给出操作数所在的地址的地址的寻址方式称为直接寻址。寻址对象：1内部数据存储器RAM，在指令中以直接地址表示；2特殊功能寄存器SFR，在指令中用寄存器名表示。

2.1.3 寄存器寻址

以通用寄存器的内容为操作数的寻址方式称为寄存器寻址。

2.1.4 寄存器间接寻址

以寄存器中的内容为地址，改地址中的内容为操作数的寻址方式称为寄存器间接寻址。

2.1.5 变址寻址

变址寻址只能对程序存储器中的数据作寻址操作。一般来说，变址寻址只有读操作没有写操作。

2.1.6 相对寻址

注意PC永远指向下一条指令的地址，所以应考虑指令是几个字节的指令

2.1.7 位寻址

2.2.2 交换指令

字节交换指令

XCH A,Rn

XCH A,dir

XCH A,@Ri

低半字节交换指令

XCHD A,@Ri ;Ri的低四位和和A的低四位交换

A的高、低半字节交换

SWAP A

2.3.2 以A为操作数的算术运算符合逻辑运算指令

ADD A,@Ri ;加法，没有进位

ADDC A,@dir ;进位加，即两数相加后还需要加上进位CY

SUBB A,Rn ;借位减

ANL A,#data ;与

ORL A,@Ri ;或

XRL A,@Ri ;异或

一个数与0异或结果不变，与1异或结果取反。

2.3.4 加1，减1指令

INC A ；自加1

DEC A ；自减1

2.3.5 十进制调整指令

DA A ；将A中二进制数相加和调整成BCD码

2.3.6 专对A的指令

CPL A ;对A取反

CLR A ;清零

RL A ;左环移

RR A ;右环移

RLC A ;左大环移

RRC A ;右大环移

2.3.7 乘、除法指令

MUL AB ；A*B->BA

16位积的高8位在B中，低8位在A中

DIV AB ；A/B->商在A中，余数在B中

2.4.1 调用程序和返回类指令

LCALL addr16 ;addr16->PC0~15

ACALL addr11 ;addr11->PC0~10

其中长调用为3字节指令，短调用为2字节指令

执行长调用或短调用指令时，首先要求出下一条指令的首地址即As（调用指令首地址）+3（调用指令长度）->PC

然后进行入栈操作，入栈是先加后压：SP+1->SP,PC0-7->SP,SP+1->SP,PC8-15->SP

最后PC=addr16

子程序返回指令

RET ；从栈顶弹出断点到PC，从子程序返回主程序。是入栈的反操作

RETI ；从中断服务程序返回

2.4.2 转移指令

无条件转移指令

短转移

AJMP addr11 ;addr11->PC0-10

长转移

LJMP addr16 ;addr16->PC0-15

间接转移

JMP @A+DPTR ；A+DPTR->PC

无条件相对转移

SJMP rel ;PC+rel->PC

条件转移指令

累加器为零，非零转移

JZ rel ;A=0转移（PC+rel ->PC）,A != 0 程序顺序执行

JNZ rel ;对上面的程序取反

减一非零转移

DJNZ Rn，rel ；Rn = Rn - 1，然后判断Rn是否为0，为0转移，不为零顺序执行

改指令有自动减1的功能

比较转移

CJNE A,dir,rel ;比较A和dir是否相等，相等则顺序执行，不相等则转移

空操作指令

NOP ；用来产生一个机器周期的延时

2.5 位操作指令

CLR bit ;位清零

SETB bit ；置1

CPL bit ；位取反

ANL C,bit ；位与

ORL C,bit ；位或

MOV C,bit ;位传送

JC rel ；CY=1，则转移（PC+rel->PC），否则顺序执行

JNC rel ;上一步的取反操作

JB bit，rel ；bit=1则转移（PC+rel->PC），否则顺序执行

JNB bit，rel ；上一步的取反操作

乱七八想

《让子弹飞》对老三的分析

​ 最近又看了两遍《让子弹飞》，姜文拍的电影确实好看。大家都知道最后老三和花姐在一起了，放弃继续追随大哥张麻子而是带着其他的弟兄们一起去了上海。知道这个结局，再看前面情节和老三的一系列行为，能够看出处处都是伏笔。

​ 来梳理一下关于老三和花姐的一些情节。他们第一次见面是在进城的时候，花姐带头敲鼓迎接县长。进城之后张麻子枪毙假麻匪，花姐则对着空气吹羽毛（应该就是鹅毛），老三对老二说：“你说她怎么不害怕呀？” ”可能是傻吧。“六子的葬礼上，老三的发言相比其他人很特别”大哥不想让大家拼命，命都不拼，还算麻匪吗？大哥不应该听他的，姓汤的不是个好玩意。“夫人的葬礼上，老三领头假装“麻匪”绑架黄四郎和两大家族，戴的是张麻子的九筒，大哥发现后呵斥“你装模作样地戴我的九筒干什么？别把事儿给我办砸了。”第一次发钱，二哥和三哥经过花姐住的屋子，把剩下的钱全部扔进花姐的窗户，不料被花姐发现并摘下他俩的面具。由于身份被识破，必须要带回来给大哥处理，所以导致了张麻子和花姐的第一次面对面接触。花姐晕过去后肯定被放走了，这样其实算是花姐欠了老二老三以及张麻子一个人情。

​ 所以后来老二老三又来到花姐的屋子一起玩Bicycle，被黄四郎发现后花姐直接上演一场美救英雄。这样一来一回自然而然就产生了感情。后面花姐一手拿枪对着自己，另一手对着张麻子的名场面“老二不辞而别，老三又要走了，我们活活的被你拆散了。“从这话和张麻子身后只有老三没有拿枪指着花姐很轻易分析出这是老三给花姐出的主意。这里同时也是暗示后面老三和花姐去上海的结局。剿匪张麻子杀了假麻子，回来后发现黄四郎替身和花姐都失踪了，老三主动说”我去找花姐。“假黄四郎和花姐为什么会同时失踪？后面的故事给了我们答案。三天后，花姐和替身被找回来，张麻子问怎么回事儿？两人的回答非常有意思，花姐：”他跑我就追，他跑我就追，就越跑越远了。“老三：“他跑我就堵，他跑我就堵就，就给堵回来了。”“就这么简单？”“就这么简单。”我们先假设杨万里（假黄四郎）真的从有枪的花姐手上跑了，他会跑向哪里？如果我是他，我肯定会选择跑回黄四郎那里。三天时间跑不回去吗?就算不讨论杨万里怎么跑的，单从花姐和老三的话就挺假的。不可能追的时候还压着速度吧哈哈哈。我觉得几乎可以肯定是老三安排花姐和杨万里的失踪，这样不仅给他和花姐三天独处的时间，而且也可以给观众甚至张麻子制造一种错觉，那就是在最后的危机关头他们把人给找回来了，立了大功，花姐还可以还了张麻子的人情。这也就很好的解释了问什么非得是老三主动说我去找花姐，因为只有他才能找到。说实话，这里觉得老三真的是厉害，虽然这里面有明显的逻辑问题，但是在那种情况下，即使是精明的张麻子也很难看出破绽，更多的则是人被找回来和即将成功的喜悦。当然，还有一种解释也可以说得通，就是张麻子在第五层，早就看穿了老三和花姐，只是没有说破。不管是哪一种，张麻子都很清楚自己想要的是什么，他注定走的是一条万分艰难的道路。

​ 抛开花姐，老三一直是大哥张麻子最信任和放心的人，在大哥不在的时候基本上都是老三指挥。给汤师爷外面老婆钱的时候，两颗砖石也是在老三手里。老三很有能力，也非常有自己的思想，是一个天生的领袖。离开张麻子另立山头的想法肯定很早之前就存在过老三的脑中，只是出于萌芽阶段，那么花姐的到来则是浇醒了这颗萌芽，坚定了离开张麻子的想法。

​ 影片最后，老三一行准备不辞而别，但敬重大哥和忠诚的老七还是给张麻子通了气。见面老三便说，还回山里啊，您腿脚都不利索了。还承诺要替二哥娶花姐，这句话跟前面回答怎么追回假黄四郎的话一样，说得像真的一样，但怎么可能？二哥说过他喜欢男人，而且这个男人肯定就是老三了（要不然不会和老二一起去和花姐一起bicycle）。不愧是姜文，不管是人物还是情节处理上，都无比精彩。
