​ 并查集用一个数组来实现

​ father[i] 表示第i个元素的根节点。

并查集需要先初始化数组

查找操作，查找操作就是对于给定的结点寻找其根节点的过程

合并操作

上面的合并操作是将两个集合的根节点连在了一起，查找起来效率非常低，采取将当前查询结点的路径上的所有结点的父节点都指向根节点，可以降低查询时的时间复杂度。

等边三角形

蒜头君手上有一些小木棍，它们长短不一，蒜头君想用这些木棍拼出一个等边三角形，并且每根木棍都要用到。 例如，蒜头君手上有长度为 1，2，3，3 的4根木棍，他可以让长度为1，2 的木棍组成一条边，另外 2 跟分别组成 2 条边，拼成一个边长为 3 的等边三角形。蒜头君希望你提前告诉他能不能拼出来，免得白费功夫。

输入格式

首先输入一个整数 n(3≤n≤20)，表示木棍数量，接下来输入 n 根木棍的长度pi (1≤pi≤10000)。

输出格式

如果蒜头君能拼出等边三角形，输出”yes”，否则输出”no”。

样例输入

4
1 2 3 3

样例输出

yes

样例输入

4
1 1 1 1

样例输出

no

思路是把数组中的值从前往后开始分配给三个边，开始三个边的长度设为0。

然后对每条边开始分配，用深度优先搜索递归，到每一条边的长度超过总长度的三分之一就返回上一层，接着对第二条边开始分配。判断成功的条件为当两条边的长度都等于总长度的三分之一，则推出一定能够组成等边三角形，ans = true。

C++常用STL之map

C++中常用的映射是map,映射只能多对一不能一对多。有点像Python里面的字典类型，有key和value。

引用

#include

using namespace std;

构造一个映射

map<T1,T2> m; //定义一个名为m的从T1类型到T2类型的映射

插入元素

插入元素需要插入一个新的映射，需要引入头文件utility

如果插入的key已经存在，则不会插入新的value代替原来的value，也就是这次插入是无效的。

访问映射可以用中括号的方式访问或者更改，这点也跟Python的字典一样

dict[“tom”]则会返回1

遍历映射也需要通过迭代器

map<string,int>::iterator it；

在遍历过程中map也是按照关键字从大到小遍历的，跟set一样

练习：

给定一串数字，统计出现次数最多的数字是多少，如果有出现次数一样多的情况则按照最大的数字输出。

不用map，首先将字符串排序之后统计每一个数连续出现了多少。

用map,先定义一个由int到int的 map，第一个int表示出现的数字，第二个表示出现该数字的个数。

C++常用STL之set

C++中常用的集合是set

引用库

#include

#using namespace std;

定义集合

set s //定义一个名为s、存储类型为T类型数据的集合

插入元素

s.insert(value) //

注意如果集合中已经存在了某个元素，再次插入不会产生效果，集合中不会出现重复元素

删除元素

s.erase(value) //如果集合中不存在这个元素，不进行任何操作

判断元素是否存在

s.count(value) //如果存在则返回1，不存在返回0

返回元素个数

s.size()

遍历集合

遍历set需要使用迭代器，固定格式为

for (set::iterator it = s.begin(); it != s.end(); it++){
cout << *it << “ “;
}

当set配合结构体来使用时，系统不知道结构体比较大小的方式额，而set是需要经过排序的，所以我们需要一种方式来告诉系统怎么比较结构体的大小。其中有一种方法叫运算符重载，需要重新定义小于符号。

bool operator<(const struct_name &rhs) const

输入

6
5 6
1 2
2 1
3 4
1 2
1 1

输出

1 1
1 2
2 1
3 4
5 6

可以看到经过遍历之后是已经自动排好序了。
